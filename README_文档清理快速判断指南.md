# 【先看这里】文档清理快速判断指南（删除前 3 分钟）

> 目标：让你以后看到任何报告/计划/调研文件时，能在 1~3 分钟内判断“删 / 归档 / 保留”。
>
> 适用范围：`docs/`、`test_report/`、`tasks/`、`outputs/`、`.tmp/` 以及类似历史沉淀目录。

---

## 一、10 秒快速判断（先看这个）

### 直接删除（高置信）
同时满足以下任意一条，可直接删：
1. **生成产物**（可重跑得到），如 `outputs/*`、`.tmp/*`、`build/`、`dist/`。
2. **空文件**（无内容、无链接引用）。
3. **重复文件**（中英文或多版本内容重复，仅保留一份）。

### 先归档再观察（中置信）
满足以下特征，优先归档（不是立刻删）：
1. **任务过程文档**（`tasks/**` 下大量 `index / plan / execution / review`）。
2. **阶段性实现计划**（`docs/plans/` 按日期命名的实现文档）。
3. **大体量测试原始包**（逐案例日志、log bundle）。

### 优先保留（低风险保留）
满足以下任意一条，不建议删：
1. 被 README / workflow / 协议文档显式引用。
2. 属于协议、规范、入口文档（SSOT、checklist、workflow）。
3. 仍可作为定位问题根因的基线报告（benchmark、审计报告）。

---

## 二、六档决策表（本次审阅沉淀）

| 档位 | 处理建议 | 典型样例 |
|---|---|---|
| 推荐清理 | 直接删除 | `outputs/*`、`.tmp/*`、`build/`、`dist/`、空文件、重复文件 |
| 可能有用 | 先归档，观察 1~2 周 | `tasks/**` 已完成任务包、`docs/plans/*.md` 历史计划、`test_report/log_bundle_*` |
| 不建议删除 | 默认保留 | `bench/cases/**`、关键研究文档（如 embedding/file-awareness/pre-router） |
| 最好保留 | 长期保留 | 全局 benchmark/阶段总结、可复用分析模板 |
| 保留 | 强制保留 | 协议 SSOT、review checklist、workflow 入口文档 |
| 需进一步研究 | 先不删，查引用再定 | 主题接近、内容重叠但不完全重复的“双份审计/双份报告” |

---

## 三、可执行的 3 步决策流程

1. **看文件属性**：是不是“生成物 / 空文件 / 重复文件”。
   - 是：直接删。
2. **看是否被引用**：README、workflow、脚本是否引用该文件或目录。
   - 有引用：保留。
3. **看重建成本**：是否可通过测试或脚本低成本重建。
   - 可重建：归档或删；不可重建：保留。

---

## 四、避免误删的红线

以下内容默认不删（除非你明确做架构迁移）：
- 协议与规范：如 `docs/skillpack_protocol.md`。
- 审阅与治理清单：如 `docs/review_checklist.md`。
- 自动化工作流入口：如 `.windsurf/workflows/*.md`。
- 用例定义与可执行基准：如 `bench/cases/*.json`。

---

## 五、目录级默认策略（可长期复用）

- `outputs/`、`.tmp/`：**定期全清**（仅保留当周正在分析的目录）。
- `tasks/`：**默认归档或清理**（只保留进行中任务）。
- `docs/plans/`：**只保留仍被执行的计划 + 少量里程碑方案**。
- `test_report/`：**保留“结论性报告”，清理“原始日志包/重复稿”**。

---

## 六、建议的文件命名与存放规范（防止以后再乱）

1. 结论性文档：`docs/analysis/YYYY-MM-DD-<topic>-report.md`
2. 一次性过程文档：放 `tasks/<task_name>/`，任务结束后归档或删除。
3. 测试原始产物：只放 `outputs/`，不放 `docs/`。
4. 中英文双份文档：保留一个主版本，另一个用链接指向主版本。

---

## 七、每次清理前的最小检查清单

- [ ] 是否为可重建生成物？
- [ ] 是否为空文件或重复文件？
- [ ] 是否被 README/workflow/脚本显式引用？
- [ ] 删除后是否影响他人快速理解当前架构？
- [ ] 如不确定，是否先归档而非直接删除？

---

最后原则：**优先删“产物”，谨慎删“知识”，绝不删“协议入口”。**
